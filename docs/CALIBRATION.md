# Система калибровки RealEngine

## Обзор

Система калибровки позволяет точно настроить параметры под **конкретный автомобиль** без необходимости угадывать заводские значения.

## 3 режима калибровки

| Режим | Параметр | Метод | Точность |
|-------|----------|-------|----------|
| **VSS** | `pulsesPerMeter` | Проехать известное расстояние | ±0.5% |
| **Injector Flow** | `injectorFlow` | Заправить полный бак, проехать | ±2% |
| **Dead Time** | `DEAD_TIME_US` | Работа на холостых, замер расхода | ±50 мкс |

## 1. Калибровка VSS (датчик скорости)

### Цель
Определить точное количество импульсов датчика скорости на **1 метр пути**.

### Процедура

#### Шаг 1: Старт калибровки
Android отправляет:
```json
{"command": "calibrate_speed_start", "msg_id": 1}
```

ESP32:
- Сбрасывает счётчик PCNT
- Начинает подсчёт импульсов VSS

#### Шаг 2: Проехать известное расстояние
- Вариант A: Proехать ровно **1 км** по одометру
- Вариант B: Proехать расстояние между двумя known точками (например, 500 м)

#### Шаг 3: Завершение калибровки
Android отправляет:
```json
{"command": "calibrate_speed_end", "data": 1000, "msg_id": 2}
```
Где `1000` = расстояние в **метрах**.

ESP32:
- Читает текущий счётчик PCNT
- Рассчитывает: `pulsesPerMeter = total_pulses / distance_meters`
- Сохраняет в NVS через SettingsPack

### Пример расчёта
```
Проехали: 1000 м (1 км)
PCNT насчитал: 2548 импульсов
pulsesPerMeter = 2548 / 1000 = 2.548 имп/м
Или: 2548 имп/км
```

### Результат
- Точная скорость без погрешности
- Точный одометр

## 2. Калибровка форсунки (Injector Flow)

### Цель
Определить реальную производительность форсунок в **мл/мин**.

### Процедура

#### Шаг 1: Заправить полный бак
- Заехать на заправку
- Заправить **до полного** (до отсечки)
- Android: `{"command": "full_tank", "msg_id": 3}`

#### Шаг 2: Старт калибровки
Android отправляет:
```json
{"command": "calibrate_injector_start", "msg_id": 4}
```

ESP32:
- Сбрасывает счётчики впрыска
- Сбрасывает `fuelUsedAccum = 0`
- Начинает накопление `calInjectorTotalUs` и `calInjectorPulses`

#### Шаг 3: Проехать 50-100 км
- Обычная езда (город/трасса)
- Чем больше расстояние — тем точнее

#### Шаг 4: Заправить снова до полного
- Заехать на **ту же** заправку
- Заправить **до полного** (до отсечки)
- Запомнить: сколько литров вошло (например, 8.52 л)

#### Шаг 5: Завершение калибровки
Android отправляет:
```json
{"command": "calibrate_injector_end", "data": 852, "msg_id": 5}
```
Где `852` = **8.52 литра × 100** (для точности).

ESP32:
- Рассчитывает: `injectorFlow = (realFuelLiters × 1000 / totalSec × 60) / injectorCount`
- Сохраняет в NVS

### Пример расчёта
```
Реальный расход: 8.52 л
Общее время впрыска: 1234.5 сек
Количество форсунок: 4
injectorFlow = (8520 / 1234.5 × 60) / 4 = 103.6 мл/мин
```

### Результат
- Точный расход топлива (±2%)
- Точный уровень топлива в баке

## 3. Калибровка Dead Time (мёртвая зона форсунки)

### Цель
Определить время задержки открытия форсунки (лаг между подачей сигнала и реальным впрыском).

### Почему это важно
- Без учёта dead time расход на **холостых завышен в 1.5-2 раза**
- Типичные значения: 600-900 мкс при 13.8В
- Зависит от напряжения бортсети

### Процедура

#### Шаг 1: Прогрев двигателя
- Прогреть двигатель до рабочей температуры (90°C)
- Выключить все потребители (кондиционер, фары, обогревы)

#### Шаг 2: Старт калибровки
Android отправляет:
```json
{"command": "calibrate_deadtime_start", "msg_id": 6}
```

ESP32:
- Начинает накопление `calDeadTimeTotalUs`
- Начинает подсчёт `calInjectorPulses`

#### Шаг 3: Работа на холостых 5-10 минут
- Двигатель работает на холостых (~750 об/мин)
- **НЕ нажимать газ!**
- Замерить реальный расход одним из способов:
  - **Вариант A**: Подключить расходомер к топливной магистрали
  - **Вариант B**: Замерить время работы и количество потраченного топлива (слив+взвешивание)

#### Шаг 4: Завершение калибровки
Android отправляет:
```json
{"command": "calibrate_deadtime_end", "data": 45, "msg_id": 7}
```
Где `45` = **45 мл/мин** (реальный расход на холостых).

ESP32:
- Рассчитывает корректировку `DEAD_TIME_US`
- Выводит рекомендуемое значение

### Пример расчёта
```
Холостые: 750 об/мин
Реальный расход: 45 мл/мин
Расчётный расход (без dead time): 70 мл/мин
Ошибка: (70 - 45) / 70 = 35%
Новый DEAD_TIME_US = 750 × 1.35 = 1012 мкс
```

### Результат
- Точный расход на холостых (0.7-1.0 л/ч)
- Компенсация изменения напряжения бортсети

## JSON-команды (полный список)

| Команда | Направление | Описание |
|---------|-------------|----------|
| `calibrate_speed_start` | Android → ESP32 | Старт калибровки VSS |
| `calibrate_speed_end` | Android → ESP32 | Завершение (data = расстояние в м) |
| `calibrate_injector_start` | Android → ESP32 | Старт калибровки форсунки |
| `calibrate_injector_end` | Android → ESP32 | Завершение (data = литры × 100) |
| `calibrate_deadtime_start` | Android → ESP32 | Старт калибровки dead time |
| `calibrate_deadtime_end` | Android → ESP32 | Завершение (data = мл/мин) |

## Хранение настроек

Все калиброванные параметры сохраняются в **NVS** через `SettingsPack`:

```cpp
struct SettingsPack {
    float   tank_capacity;     // Ёмкость бака (л)
    float   injector_flow;     // Производительность форсунки (мл/мин)
    uint8_t injector_count;    // Количество форсунок
    float   pulses_per_meter;  // Импульсов VSS на метр
    uint8_t kline_protocol;    // Протокол K-Line
};
```

При старте ESP32 загружает настройки из NVS → все модули получают актуальные данные.

## Рекомендации по калибровке

### Порядок выполнения
1. **VSS** — сначала (влияет на скорость и одометр)
2. **Injector Flow** — потом (влияет на расход и уровень топлива)
3. **Dead Time** — последним (тонкая настройка холостых)

### Частота калибровки
- **VSS**: 1 раз (не меняется, пока не заменён датчик)
- **Injector Flow**: 1 раз в 50-100 тыс. км (износ форсунок)
- **Dead Time**: 1 раз в год или при замене аккумулятора

### Точность
- VSS: ±0.5% (зависит от точности замера расстояния)
- Injector Flow: ±2% (зависит от точности заправки)
- Dead Time: ±50 мкс (зависит от точности замера расхода)

## Troubleshooting

### Проблема: VSS калибровка даёт странные значения
**Решение:**
- Проверить подключение VSS (оптопара PC817 или делитель)
- Убедиться, что сигнал 0-3.3В (не 5-12В!)
- Проверить PCNT: `pcnt_get_counter_value()` должен меняться при движении

### Проблема: Injector Flow получается <200 или >300 мл/мин
**Решение:**
- Проверить, что бак был **действительно** полный
- Убедиться, что `injectorCount` правильный (4 цилиндра)
- Повторить калибровку на большем расстоянии (100+ км)

### Проблема: Dead Time получается <400 или >1200 мкс
**Решение:**
- Проверить, что двигатель был **прогрет** (90°C)
- Убедиться, что холостые стабильны (750±50 об/мин)
- Проверить напряжение бортсети (должно быть 13.8-14.4В)
