# K-Line Research — Toyota RAV4 2004

> **Дата исследования:** 2026-04-05  
> **Цель:** Реализация K-Line диагностики Toyota RAV4 2004 через MC33290 + ESP32  
> **Статус:** Сбор информации завершён

---

## 1. Оборудование

### MC33290 — K-Line трансивер
- **Производитель:** NXP Semiconductors
- **Стандарт:** ISO 9141-2 (физический уровень)
- **Корпус:** SOIC-8
- **Питание:** 5V или 3.3V (совместим с ESP32)

#### Подключение к ESP32:
```
MC33290          ESP32
──────────────────────────
Pin 1 (TXD)  →   GPIO 16 (UART2 RX)
Pin 2 (RXD)  →   GPIO 17 (UART2 TX)
Pin 3 (GND)  →   GND
Pin 4 (VCC)  →   3.3V или 5V
Pin 5 (K-Line) →   OBD-II Pin 7 (через резистор 510Ω к +12V)
Pin 6 (L-Line) →   Не используется (опционально)
Pin 7 (WAKE)   →   Не используется
Pin 8 (INH)    →   Не используется
```

#### Схема подключения K-Line к OBD-II:
```
OBD-II Pin 7 ──[510Ω]───+── K-Line (MC33290 Pin 5)
                         │
                        [1kΩ]
                         │
                       +12V (авто)
```

## 2. Протокол связи

### ISO 9141-2 (K-Line)

#### Инициализация (5-baud Slow Init):
1. **Ждём** ≥ 300мс после включения зажигания
2. **Tx LOW** — 25мс (старт-бит)
3. **Tx адрес** — 5 бит со скоростью 5 бод (≈200мс на бит):
   - Адрес `0x33` ( broadcast для ECM)
   - Биты: 1-1-0-1-1 (LSB first, без стоп-бита)
4. **Ждём ответ** от ECU через 25мс (W1 = 25мс)
5. **Ключевые байты** (ECU → Tester):
   - `0x8F` (или `0xEF` для TCM) — подтверждение
   - `0x8F` — повтор
   - `0x01` — идентификатор ECU (ECM)
   - `0x08` — идентификатор ECU (TCM)
6. **Переход на 10400 baud** — обычная UART связь

#### Fast Init (альтернатива):
- 25мс LOW на K-Line → ECU отвечает `0x8F 0x8F 0x01`
- Работает не на всех Toyota 2004

#### Параметры связи после инициализации:
- **Baud rate:** 10400 бит/с
- **Data bits:** 8
- **Stop bits:** 1
- **Parity:** None

## 3. Адреса блоков управления (ECU)

| Блок | Адрес (Hex) | Идентификатор (Init) | Описание |
|------|-------------|----------------------|----------|
| **ECM** (Engine) | `0x7E0` | `0x01` | Двигатель |
| **TCM** (Transmission) | `0x7E1` | `0x08` | Автоматическая коробка |
| **ABS** | `0x7E3` | `0x0A` | Антиблокировочная система |
| **SRS** (Airbag) | `0x7E4` | `0x0B` | Подушки безопасности |
| **Body** | `0x7E5` | `0x0C` | Кузовная электроника |
| **Combination Meter** | `0x7E6` | `0x0D` | Приборная панель |

## 4. Формат кадров

### OBD2 Standard Frame (Mode 01):
```
[Длина] [Режим] [PID] [Контрольная сумма]
  02     01     0C        CS
```

### Toyota Mode 21 (Extended):
```
[Длина] [Режим 21] [PID] [CS]
  02     21        XX     CS
```

### Multi-frame Response (ISO-TP):
- **First Frame (FF):** `[0x10 + длина_H] [длина_L] [Данные...]`
- **Flow Control (FC):** Tester → ECU: `30 00 00` (Go, BlockSize=0, STmin=0)
- **Consecutive Frame (CF):** `[0x21] [Данные...]` (инкремент sequence)

### 5.5 Mode 21 PID `21 52` — Данные трансмиссии

Ответ: `61 52 [A B C D E F ...]`

| Байт | Параметр | Формула | Единицы |
|------|----------|---------|---------|
| A | Соленоид SLT (давление) | `A * 0.05` | V |

| C | Соленоид S2 | `C` (0/1) | — |
| D | Соленоид SLU | `D * 0.05` | V |
| E | Обороты входного вала | `(E*256+F) / 4` | об/мин |
| F | Обороты выходного вала | `(G*256+H) / 4` | об/мин |

### 5.6 Mode 02 — Freeze Frame данные
- `02 01 XX` — те же PID что и Mode 01, но значения на момент ошибки
- Используется для диагностики

### 5.7 Mode 03 — Чтение подтверждённых DTC
- **Команда:** `03`
- **Ответ ECM:** `43 [Код1_H Код1_L Код2_H Код2_L ...]`
- **Формат DTC:** `P0XXX` где:
  - Первый символ: `P` (Powertrain), `C` (Chassis), `B` (Body), `U` (Network)
  - Первая цифра: 0=SAE, 1=Manufacturer
  - Пример: `P0300` = Random Misfire

#### Расширенное чтение DTC (все блоки):
- **Команда:** `18 83 02 FF FF` — прочитать из ВСЕХ блоков
- **Ответ:** `[Кол-во DTC] [Код1] [Код2] ...`

#### Чтение DTC по блокам:
- **ECM:** `18 83 02 00 01` — только двигатель
- **TCM:** `18 83 02 00 02` — только АКПП
- **ABS:** `18 83 02 00 03` — только ABS
- **SRS:** `18 83 02 00 04` — только подушки

### 5.8 Mode 04 — Сброс DTC и MIL
- **Команда:** `04`
- **Ответ:** `44` (подтверждение)
- Очищает MIL, подтверждённые DTC, Freeze Frame

### 5.9 Mode 07 — Pending DTC
- **Команда:** `07`
- Показывает ошибки, которые ещё не подтверждены (pending)

### 5.10 Mode 0A — Permanent DTC
- **Команда:** `0A`
- Показывает "застывшие" ошибки (permanent), которые нельзя сбросить

## 6. Сброс адаптации АКПП

### 6.1 Через K-Line диагностические команды

#### Вариант 1: Через TCM (режим 31 — Active Test)
```
Запрос: 31 01 02 XX (XX = локальный ID теста)
Ответ: 71 01 YY (YY = статус выполнения)
```

**Известные ID для Toyota:**
- `31 01 01 01` — Reset Memory ECM
- `31 01 01 02` — Reset Memory TCM (адаптация АКПП)

**Полная последовательность:**
1. Подключиться к TCM (адрес `0x7E1`)
2. Отправить: `10 10` (Start Diagnostic Session — Extended)
3. Отправить: `31 01 01 02` (Reset TCM Adaptation)
4. Получить: `71 01 01 02` (подтверждение)
5. Перезагрузить TCM: `11 01` (ECU Reset)

#### Вариант 2: Через ECM (Mode 21 + запись)
```
Запись (Write Data By Local Identifier):
2E [Local ID] [Значение]
```

### 6.2 Ручной метод (без диагностики)
1. Включить зажигание (ON), НЕ запускать двигатель
2. Нажать педаль газа полностью
3. Подождать 30 секунд
4. Отпустить педаль
5. Выключить зажигание
6. Подождать 10 секунд
7. Завести двигатель, дать поработать на ХХ 5 минут

### 6.3 Через отключение питания
1. Отключить минусовую клемму аккумулятора
2. Подождать ≥ 10 минут (разряд конденсаторов)
3. Подключить обратно
4. Завести двигатель
5. Проехать 30 минут в обычном режиме

## 7. Прокачка ABS через K-Line

### 7.1 Через Techstream / Car Scanner
Techstream выполняет: **Chassis → ABS-VSC-TRC → Active Test → Perform Actuator Pattern**

### 7.2 K-Line команды

#### Последовательность ABS Bleed:
1. Подключиться к ABS блоку (адрес `0x7E3`)
2. Start Diagnostic Session (Extended): `10 83` или `10 90`
3. Diagnostic Mode: `27 [Seed & Key]` (если требуется)
4. Active Test Start: `31 01 02 XX` (XX = ID теста ABS actuator)
5. Следовать инструкции Techstream
6. Active Test Stop: `31 02 02 XX`

#### Известные ID для ABS (Toyota):
- `31 01 02 01` — ABS Actuator Test (прокачка)
- `31 01 02 02` — ABS Motor Run
- `31 01 02 03` — ABS Solenoid Test

#### Ответы:
- `71 01 02 01` — тест запущен
- `71 02 02 01` — тест завершён
- `7F 31 XX` — ошибка (XX = код ошибки)

### 7.3 Через Car Scanner ELM OBD2
Приложение Car Scanner поддерживает ABS Bleed для Toyota через:
- Профиль: `Toyota: OBD-II / EOBD ~2006-2009 CAN + extra sensors`
- Меню: **ABS → Active Test → Brake Bleed**

## 12. План реализации K-Line Task

### Архитектура:
```
┌──────────────────────────────────────────────┐
│            K-Line Task (FreeRTOS)            │
│  Priority: 2, Stack: 4096, Core: 0          │
│                                              │
│  ┌────────────────────────────────────────┐  │
│  │  MC33290 → UART2 (TX=17, RX=16)       │  │
│  │  ISO9141-2: 5-baud Slow Init          │  │
│  │  10400 baud после инициализации        │  │
│  └────────────────────────────────────────┘  │
│                     │                        │
│  ┌──────────────────▼──────────────────────┐  │
│  │  ISO-TP Frame Handler                   │  │
│  │  - Multi-frame (FF, CF, FC)             │  │
│  │  - Single-frame                          │  │
│  │  - Flow Control                          │  │
│  └─────────────────────────────────────────┘  │
│                     │                         │
│  ┌──────────────────▼──────────────────────┐  │
│  │  OBD2 Request Manager                   │  │
│  │  ┌───────────────────────────────────┐  │  │
│  │  │ Циклический опрос (500мс):       │  │  │
│  │  │ - 01 0C → RPM (ECM)              │  │  │
│  │  │ - 01 0D → Speed (ECM)            │  │  │
│  │  │ - 01 05 → Coolant Temp (ECM)     │  │  │
│  │  │ - 21 6C → ATF Temp (ECM)         │  │  │

│  │  └───────────────────────────────────┘  │  │
│  │  ┌───────────────────────────────────┐  │  │
│  │  │ По запросу (DataBus):            │  │  │
```

## 13. Известные проблемы и нюансы

1. **K-Line на RAV4 2004** может использовать **broadcast init** (`0x33`) или **адресную инициализацию** (`0x7E0`)
2. **Multi-frame ответы** — PID `21 D9` и `21 81` возвращают >7 байт, нужен ISO-TP
3. **TCM на 2004** может быть интегрирован в ECM (не отдельный блок) — проверять
4. **ABS Bleed** может не работать на ранних моделях — нужна проверка на конкретном авто
5. **Baud rate 10400** — нестандартный, UART ESP32 поддерживает (делитель 80000000/10400 = 7692)
6. **WAKE pin MC33290** — можно не подключать, инициализация через K-Line
7. **L-Line** — не используется для большинства диагностических операций

- [ ] Зажигание ON (двигатель не обязательно)
