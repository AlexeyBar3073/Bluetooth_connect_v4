# Схема подключения датчика уровня топлива (Toyota RAV4 2004)

## Обзор

Датчик уровня топлива Toyota RAV4 — это **два последовательных поплавка** (основной + дополнительный для компенсации наклона).

| Параметр | Значение |
|----------|----------|
| Основной поплавок (полный → пустой) | ~54 Ом |
| Дополнительный поплавок | ~53 Ом |
| **Суммарное сопротивление** | **~107 Ом** |
| Тип | Реостат (переменное сопротивление) |
| Расположение | Топливный бак (в сборе с бензонасосом) |

---

## Проблема: Влияние на штатный индикатор

Если подключить ESP32 ADC **напрямую к поплавку**, входное сопротивление ESP32 (~1 МОм) и наводки WiFi/BT заставят **штатную стрелку врать**.

**Решение:** Буферный усилитель **LM358** (повторитель напряжения).

---

## Схема подключения (окончательная)

### Полная схема с LM358

```
Поплавок (0-107 Ом) ───┬─── к приборной панели (штатный провод)
                        │
                        └───[10 кОм]───┬─── LM358 Pin 3 (+)
                                       │
                                      [100 кОм]
                                       │
                                      GND

LM358 питание:
  Pin 8 (V+) ─── +12В (через фильтр: 100 Ом + 100 мкФ)
  Pin 4 (V-) ─── GND

LM358 выход:
  Pin 1 (OUT) ───[1 кОм]───┬─── GPIO34 (ADC)
                            │
                           [100 нФ]
                            │
                           [3.3V стабилитрон] ← Защита от перенапряжения
                            │
                           GND
```

### Почему LM358 питание от +12В?

У LM358 **выход не доходит до напряжения питания** на ~1.5В:
- Питание от 5В → макс. выход = 3.5В (мало для полного диапазона)
- Питание от 12В → макс. выход = 10.5В (покрывает весь диапазон поплавка)

### Расчёт напряжения на ADC

**Делитель после LM358:**

```
Vadc = V_out_LM358 × R2 / (R1 + R2)
```

| Состояние | R_поплавка | V_LM358 | Vadc (после делителя) | ADC (12-bit) |
|-----------|-----------|---------|----------------------|--------------|
| **Полный бак** | 0 Ом | ~5.0В | **2.27В** | **2815** |
| **Полупустой** | 54 Ом | ~3.3В | **1.50В** | **1860** |
| **Пустой бак** | 107 Ом | ~2.5В | **1.14В** | **1414** |

**Делитель:** R1=1 кОм, R2=1 кОм (коэффициент 0.5)

**Диапазон:** 1.14В → 2.27В (Δ = 1.13В)  
**Уровней ADC:** 2815 - 1414 = **1401 уровней** (точность ±0.07%)

### ⚠️ Почему максимум 2.27В, а не 3.3В?

ADC ESP32 **нелинеен** на краях диапазона:
- 0-0.3В: насыщение (все значения = 0)
- 2.5-3.3В: насыщение (все значения = 4095)

**Линейная зона ADC ESP32:** 0.5В → 2.5В

Наш диапазон (1.14-2.27В) попадает **в середину линейной зоны** ✅

## Компоненты

| Компонент | Номинал | Кол-во | Назначение |
|-----------|---------|--------|-----------|
| **LM358** | DIP-8 | 1 | Буферный усилитель (повторитель) |
| **R_вх** | 10 кОм, 1% | 1 | Входной делитель (защита LM358) |
| **R_bias** | 100 кОм, 1% | 1 | Смещение входа LM358 |
| **R1** | 1 кОм, 1% | 1 | Делитель выхода LM358 |
| **R2** | 1 кОм, 1% | 1 | Делитель выхода LM358 |
| **C_f** | 100 нФ | 1 | Фильтр помех (ADC-GND) |
| **D1** | 3.3В стабилитрон (1N4728A) | 1 | Защита GPIO34 от перенапряжения |
| **R_filter** | 100 Ом | 1 | Фильтр питания LM358 |
| **C_filter** | 100 мкФ, 16В | 1 | Фильтр питания LM358 |

## Калибровка (обязательно!)

### Шаг 1: Полный бак

1. Заправить **до полного** (до отсечки)
2. Включить зажигание (не запуская двигатель)
3. Замерить напряжение на GPIO34 мультиметром:
   ```
   Vadc_full = ??? В (записать!)
   ```

### Шаг 2: Пустой бак

1. Проехать до загорания лампочки (остаток ~5-7 л)
2. Замерить напряжение на GPIO34:
   ```
   Vadc_empty = ??? В (записать!)
   ```

### Шаг 3: Обновить код

В файле `real_engine.cpp`:

```cpp
static float    fuelAdcFull     = 2.27f;   // ← Заменить на замеренное!
static float    fuelAdcEmpty    = 1.14f;   // ← Заменить на замеренное!
```

## Подключение к Toyota RAV4

### Где подключаться?

| Место | Плюсы | Минусы |
|-------|-------|--------|
| **Под сиденьем** (разъём бензонасоса) | Легко добраться, короткие провода | Дальше от ESP32 |
| **За приборкой** (разъём комбинации) | Ближе к ESP32 | Снимать приборку (30 мин) |

**Рекомендация:** Подключаться **под сиденьем**, сигнал вести через **экранированный провод** к ESP32.

### Пины разъёма бензонасоса

| Пин | Назначение | Цвет провода (типично) |
|-----|-----------|------------------------|
| **FUA** | Поплавок основной | Белый/Чёрный |
| **FU** | Поплавок дополнительный | Белый/Чёрный (другой) |
| **E2** | Масса (GND) | Коричневый |

⚠️ **Важно:** Проверить мультиметром!

## Проверка LM358 (тест на столе)

```
+12В ───[100 Ом]───┬─── LM358 Pin 8
                    │
                   [100 мкФ]
                    │
                   GND

Потенциометр 200 Ом ─── LM358 Pin 3 (+)
                       │
                      [100 кОм]
                       │
                      GND

LM358 Pin 1 (OUT) ───[1 кОм]───┬─── Мультиметр
                                │
                               [100 нФ]
                                │
                               GND
```

**Тест:**
- Крутим потенциометр на 0 Ом → Vout = ~5.0В → после делителя = 2.5В
- Крутим на 107 Ом → Vout = ~2.5В → после делителя = 1.25В

## Troubleshooting

### Проблема: ADC показывает 0В или 3.3В постоянно

**Причины:**
- LM358 не питается (проверить Pin 8 = +12В)
- Обрыв провода поплавка
- GPIO34 настроен как выход (должен быть INPUT)

**Решение:**
1. Проверить мультиметром выход LM358 (Pin 1)
2. Проверить напряжение на GPIO34 (должно быть 1-2.5В)
3. Убедиться, что `pinMode(34, INPUT)`

### Проблема: Показания плавают ±10%

**Причины:**
- Топливо плещется в баке
- Наклон автомобиля
- Помехи от WiFi/BT

**Решение:**
1. Увеличить сглаживание в `readFuelAdc()` (α = 0.1 вместо 0.25)
2. Добавить RC-фильтр (100 нФ)
3. Использовать **оба** поплавка (уже последовательно — компенсация наклона)

### Проблема: Полный бак = 0%

**Причины:**
- Перепутаны `fuelAdcFull` и `fuelAdcEmpty`
- Неправильная формула

**Решение:**
- Полный бак = **больше** Вольт (сопротивление поплавка ≈ 0)
- Пустой бак = **меньше** Вольт (сопротивление поплавка ≈ 107 Ом)
- Формула: `pct = (Vadc - fuelAdcEmpty) / (fuelAdcFull - fuelAdcEmpty) × 100%`

## Пример кода (тест вне авто)

```cpp
void setup() {
    Serial.begin(115200);
    analogReadResolution(12);
    pinMode(34, INPUT);
}

void loop() {
    int raw = analogRead(34);
    float volts = raw * (3.3 / 4095.0);
    
    // Расчёт уровня топлива (подставить свои значения!)
    float full = 2.27;   // Вольт (полный бак)
    float empty = 1.14;  // Вольт (пустой бак)
    float pct = (volts - empty) / (full - empty) * 100.0;
    pct = constrain(pct, 0, 100);
    
    Serial.printf("ADC: %d, V: %.3f, Fuel: %.1f%%\n", raw, volts, pct);
    delay(500);
}
```

**Тест с резисторами:**
```
R_поплавка = 0 Ом    → Fuel: 100%
R_поплавка = 54 Ом   → Fuel: 50%
R_поплавка = 107 Ом  → Fuel: 0%
```
